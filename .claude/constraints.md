## Key Constraints

**nexus:** custom image `infrastructure/configs/nexus/`; `provision.sh` runs bg at startup via `docker-entrypoint.sh`; idempotent via `/nexus-data/.provisioned`; repos in `repos.json` (REST API, no Groovy); DinD `command` array: no shell var expansion — `nexus:8082` hardcoded; CI `script` uses `$NEXUS_DOCKER_REGISTRY`; `NEXUS_USER/PASSWORD/NPM_TOKEN` are GitLab CI/CD secrets; Dockerfiles: `ARG REGISTRY=mcr.microsoft.com` — CI passes `--build-arg REGISTRY=$NEXUS_DOCKER_REGISTRY`.

**nginx auth:** subrequest `/internal/auth/check` → `users-service:9100/auth/check` before every request; only `/api/users/auth/` is public.

**nginx expenses routing:** `location /api/expenses` + `rewrite ^/api/expenses/?(.*)$ /$1 break` + `proxy_pass http://expenses-service:9200` strips prefix; `ExpenseController`: `[Route("")]`; all others keep prefixes; `UserConfigController`: `[Route("config")]`; admin: `[Route("admin/...")]`; `ExpenseImportController`: `[Route("import")]`.

**admin role guard (expenses):** `AppAdminAttribute` → `JwtCookieReader.GetIsAdmin(Request)` decodes `isAdmin` JWT from `auth_token` cookie or `Authorization: Bearer`; 403 if false/missing. `JwtCookieReader`: `GetUserId`+`GetIsAdmin`, both call `ExtractToken` (cookie→header fallback).

**admin role guard (users):** `AdminAuthorizeAttribute` → `isAdmin` JWT from `auth_token` cookie (same base64url decode); 403 if false/missing.

**isAdmin JWT claim (users):** `JwtTokenService.GenerateTokenAsync` queries `IRoleRepository.GetUserRolesAsync(userId)`; adds `"isAdmin":"true"` when any role `Code=="APP_ADMIN"`; `GET /auth/session` → `SessionResponse.IsAdmin:bool`; `POST /auth/login` → `LoginResponse.IsAdmin:bool`; `AuthContext.login` merges into state → admin nav renders immediately.

**APP_ADMIN role:** `Role` row `Code="APP_ADMIN"`, `Name="App Administrator"` — seed manually (no EF migration). Assign: `PUT /admin/users/{id}/roles` or direct insert into `URR_UserRoles`.

**cross-service DB:** expenses reads `ext.USR_Users` from users DB via `Repositories/External/UserRepository` (read-only, in `ExpensesDbContext`); `GetAdminUserIdsAsync()`: `USR_IsAdmin=true AND USR_IsDeleted=false`; `USR_IsAdmin` (migration `AddUserIsAdmin`) synced via outbox: `user.created`→false, `user.updated` (by `AdminUserService.SetUserRolesAsync`) carries current `IsAdmin` → `UserEventConsumer` writes via `SaveOrUpdateUserAsync`.

**messaging:** RabbitMQ singleton `IRabbitMQService.GetConnection()`; env prefixes: expenses `EXPENSES_MANAGEMENT_EXPENSES_RABBITMQ_*`, users `EXPENSES_MANAGEMENT_USERS_RABBITMQ_*`, notifications `EXPENSES_MANAGEMENT_NOTIFICATIONS_RABBITMQ_*`; vhost `expense_management`; `appsettings.json` > env vars — pattern: `GetValue("Key", envVar) ?? hardcoded`; `UserEventConsumer.ExecuteAsync` retries on `BrokerUnreachableException` every 5s; users publishes to `users.events` (keys: `user.created/updated/deleted`, `user.email.verification.requested`, `user.password.reset.requested/changed` — via outbox); expenses consumes `expenses.users.sync` (binding `user.#`) as `UserEventConsumer` BackgroundService → `SaveOrUpdateUserAsync/DeleteUserAsync`; expenses `RabbitMQService` requires `DispatchConsumersAsync=true`; expenses publishes to `expenses.events` (keys: `family.member.removed`, `family.invitation.requested/accepted`, `family.member.joined`, `family.expense.added/deleted`, `expenses.import.completed`, `expenses.rate.conflict`); notifications consumes `notifications.expenses.sync` (bindings `family.#`+`expenses.#`) as `FamilyEventConsumer` AND `notifications.users.email` (binding `user.#`) as `UserNotificationEventConsumer`.

**outbox pattern (users):** `MSG_OutboxEvents` table; `OutboxPublisherService` polls 5s, max 5 retries, calls `IUserEventPublisher.PublishRaw(eventType,payload,messageId)`; replay: `POST /messaging/replay`; stats: `GET /messaging/outbox/stats`.

**outbox pattern (expenses):** `OutboxEvents` table; `FamilyOutboxPublisherService` polls 5s/5 retries, calls `IFamilyEventPublisher.PublishRaw`; written by: `FamilyService.RemoveMemberAsync/InviteAsync/AcceptInviteAsync`, `ExpenseService.AddAsync/DeleteAsync`, `CsvImportService.ConfirmImportAsync`, `CurrencyRateService.RunDailyUpdateAsync`; all non-fatal try/catch; migration: `AddExpensesOutbox`.

**inbox pattern (expenses):** `InboxEvents` table; `UserEventConsumer` extracts `MessageId` from `ea.BasicProperties.MessageId` (fallback: GUID); `IInboxRepository.ExistsAsync` → duplicate: ack+skip; success: write `InboxEvent{Status="Processed"}` then ack; failure: nack, no write.

**inbox pattern (notifications):** same dedup as expenses; `FamilyEventConsumer`+`UserNotificationEventConsumer`; `OnMessageReceivedAsync`/`Ack`/`Nack` are `protected virtual` for `TestableConsumer`.

**notifications service:** port 9300; sole SMTP dispatcher (others don't send email); `NotificationsDbContext`: `Notifications`+`InboxEvents`; `UseIdentityAlwaysColumn()`+partial index guarded by `Database.IsNpgsql()`; `NotificationHub` (SignalR) at `/ws/notifications` — auth via `auth_token` cookie, groups by `userId`; dispatch: persist DB → hub push → email (all non-fatal); email-only handlers skip DB+hub; fan-out handlers loop per recipient; consumers: `FamilyEventConsumer` (queue `notifications.expenses.sync`, `family.#`+`expenses.#`, 8 types), `UserNotificationEventConsumer` (queue `notifications.users.email`, `user.#`, 3 types); `FamilyEventConsumer`: deserialize `BaseEventEnvelope` first, switch+deserialize specific DTO; env prefixes: `EXPENSES_MANAGEMENT_NOTIFICATIONS_DATABASE_*/_RABBITMQ_*/_EMAILAUTH_*`; nginx: `/api/notifications/ws/` (WebSocket, 3600s) + `/api/notifications` → `notifications-service:9300`; frontend: `@microsoft/signalr` `HubConnectionBuilder` + `withCredentials:true`; 6 email templates in `Assets/EmailTemplates/` (EmbeddedResource).

**backend tests (notifications):** `TestNotificationsDbContextWrapper` (SQLite+EnsureCreated); `TestableConsumer` exposes `InvokeOnMessageReceived`, overrides `Ack`/`Nack`; `NotificationHub`+`EmailHelper`: `[ExcludeFromCodeCoverage]`; 4 Npgsql-guard lines uncovered — accepted.

**DB migrations:** auto-run at startup via `MigrateAsync()`.

**OutboxEvent.Id (users):** `long`; `UsersAppDbContext.OnModelCreating`: `UseIdentityAlwaysColumn()` only when `Database.IsNpgsql()`, else `ValueGeneratedOnAdd()` (SQLite compat); `TestDbContextEnsureCreated` uses this path (never `Migrate()` for `OutboxRepositoryTests`).

**user soft-delete:** `User.IsDeleted` (bool,false) + `DeletedAt` (DateTime?); `DeleteUserAsync` sets flags, never `.Remove()`; all 5 repo query methods filter `!u.IsDeleted`; partial unique index `ux_usr_email_active` on `USR_Email WHERE USR_IsDeleted=FALSE`; `IsDisabled` = suspension, separate.

**email verification expiry:** `User.EmailValidationHashExpiresAt` (DateTime?) — `UtcNow+24h` on register/resend; `ValidateEmailAsync`: non-null+past → null → error page; null=no expiry (legacy); `UpdateEmailValidationHashAsync` atomically sets hash+expiresAt; `ResendVerificationEmailAsync` always 200 (no leak); re-registering unverified email silently triggers resend.

**rate limiting (users):** .NET 8 `Microsoft.AspNetCore.RateLimiting`, fixed-window per IP; policies: `login`(10/1m), `register`(5/10m), `resend_verification`(3/10m), `validate_email`(10/5m), `request_password_reset/change_password_reset/create_password`(5/10m), `change_password`(10/5m), `refresh`(20/1m), `messaging_replay`(5/1m); no limit on `GET /auth/check`, `/auth/session`, `POST /auth/logout`, `GET /messaging/outbox/stats`.

**rate limiting (expenses):** `expenses_global` sliding window 100req/60s, 6 segments, per IP; class-level `[EnableRateLimiting]` on `CategoryController`, `CurrencyController`, `ExpenseController`.

**family system (expenses):** `FamilyService.CreateDefaultAsync(userId)` idempotent (checks `HasDefaultFamilyAsync`); called by `UserEventConsumer` on `user.created`; `FamilyInvitation` table (GUID token); expiry: `FamilyOptions.InviteExpiryInDays` (env `EXPENSES_MANAGEMENT_EXPENSES_FAMILY_INVITE_EXPIRY_IN_DAYS`, default 7); invite URL: `FamilyOptions.InviteBaseUrl` + `?token={token}`; `AcceptInviteAsync`: validates not expired/accepted + email matches `InviteeEmail`; `ArchiveAsync` = soft-delete, non-default only; `RemoveMemberAsync` also calls `RemoveMemberAttributionsAsync`; expense `FamilyIds int[]?`: null→default family, empty→no attribution, provided→validate membership; exceptions: `FamilyForbiddenException`(403), `FamilyNotFoundException`(404), `FamilyConflictException`(409), `FamilyInvitationException`(400); role→ID via `ILookupCacheService.GetIdAsync<FamilyRole>` (title-case); name uniqueness: `ExistsWithNameForUserAsync` case-insensitive across non-deleted memberships → `FamilyConflictException("FAMILY_NAME_ALREADY_EXISTS")`→409; frontend handles 409 via `setError('name',…)`; `GetPendingInvitationsByFamilyAsync`: filters `AcceptedAt==null && ExpiresAt>UtcNow`; `RevokeInvitationAsync`: Head-only, hard-deletes `FamilyInvitation` row (no outbox event); both behind Head role check via `ILookupCacheService.GetIdAsync<FamilyRole>(RoleHead)`.

**family system (expenses service):** `IUserRepository.GetUserByEmailAsync` for invite flow; `GetUserByIdAsync` filters `!u.IsDeleted`; `IFamilyRepository` injected into `ExpenseService`; `GetAdminUserIdsAsync()` for `RateConflictEventMessage.AdminUserIds`.

**tag system:** `Tags` global — no per-user ownership; unique `Name` (case-sensitive); `UserTags(UserId,TagId)` tracks adoption; adoption via `POST /tags` or auto-adopt on attach; `ITagRepository.GetFamilyAsync` returns co-member tags not already own; `IsVisibleAsync` = own OR co-member `UserTag`; `FamilyForbiddenException`(403) for invisible; `WriteExpenseTagsAsync`: validate→auto-adopt→insert `ExpenseTag`; `MapToDto` takes `explicitTags?` (add/update), falls back to `e.ExpenseTags` (read); Moq CS0854: pass `It.IsAny<string>()` in test Verify for optional tag params.

**category name uniqueness:** `ICategoryRepository.ExistsWithNameAsync(name,parentCategoryId,excludeId?)` case-insensitive per level; `AdminCategoryService` throws `InvalidOperationException("CATEGORY_NAME_DUPLICATE")`→400; scope: per-level only.

**expenses soft-delete (`Category`, `Family`, `Expense`):** all use `IsDeleted`+`DeletedAt`; `SoftDeleteAsync` sets flags, never `.Remove()`; raw SQL migrations use `CURRENT_TIMESTAMP` (not `NOW()`) for SQLite compat.

**backend tests:** `TestExpensesDbContextWrapper` (in-memory, Migrate); `TestExpensesDbContextEnsureCreated` (SQLite+EnsureCreated, `OutboxEvents` `long` PK — `Database.IsNpgsql()` guard + `ValueGeneratedOnAdd()`, migration omits `type:"bigint"`); `TestDbContextWrapper` (SQLite+Migrate, users, int/short PKs); `TestDbContextEnsureCreated` (SQLite+EnsureCreated, users, `OutboxEvent` `long` PK); `TestNotificationsDbContextWrapper` (SQLite+EnsureCreated, notifications); never mock DbContext.
