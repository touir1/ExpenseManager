## Key Constraints

Non-obvious things Claude cannot derive from reading files:

**nginx auth:** subrequest `/internal/auth/check` → `users-service:9100/auth/check` before every request; only `/api/users/auth/` is public.
**cross-service DB:** expenses reads `ext.USR_Users` from users DB via `Repositories/External/UserRepository` (read-only, mapped in `ExpensesDbContext`).
**messaging:** RabbitMQ singleton `IRabbitMQService.GetConnection()`; expenses config env prefix: `EXPENSES_MANAGEMENT_EXPENSES_RABBITMQ_*`, users config prefix: `EXPENSES_MANAGEMENT_USERS_RABBITMQ_*`; users service publishes to `users.events` topic exchange (routing key = `user.created` / `user.updated` / `user.deleted`); expenses service consumes queue `expenses.users.sync` (binding `user.#`) as `UserEventConsumer` BackgroundService → calls `IUserRepository.SaveOrUpdateUserAsync` or `DeleteUserAsync`; trigger: `RegistrationService.ValidateEmailAsync` calls `IUserRepository.ValidateEmailAsync` (saves user), then independently calls `IOutboxRepository.EnqueueAsync` to write the outbox event — no shared transaction; expenses `RabbitMQService` requires `DispatchConsumersAsync = true` for async consumer.
**outbox pattern (users):** `MSG_OutboxEvents` table stores unpublished events; `OutboxPublisherService` (BackgroundService) polls every 5 s, max 5 retries, calls `IUserEventPublisher.PublishRaw(eventType, payload, messageId)`; replay via `POST /messaging/replay?eventType=&from=&forceAll=`; stats via `GET /messaging/outbox/stats`.
**inbox pattern (expenses):** `InboxEvents` table deduplicates consumed messages; `UserEventConsumer` extracts `MessageId` from `ea.BasicProperties.MessageId` (fallback: new GUID); checks `IInboxRepository.ExistsAsync` before processing — duplicate → ack and skip; on success writes `InboxEvent { Status="Processed" }` then acks; on failure nacks without inbox write.
**DB migrations:** auto-run at startup via `MigrateAsync()`.
**OutboxEvent.Id (users) — provider-specific config:** `OutboxEvent.Id` is `long`; `UsersAppDbContext.OnModelCreating` applies `UseIdentityAlwaysColumn()` only when `Database.IsNpgsql()`, and `ValueGeneratedOnAdd()` otherwise — this lets `EnsureCreated()` on SQLite emit `INTEGER PRIMARY KEY` (auto-increment) for tests; Npgsql's `IdentityAlwaysColumn` annotation forces `BIGINT` even on SQLite, breaking test auto-increment; `TestDbContextEnsureCreated` uses this path (never call `Migrate()` for `OutboxRepositoryTests`).
**backend tests:** use `TestExpensesDbContextWrapper` (in-memory DB, expenses); `TestDbContextWrapper` (SQLite + Migrate, users — for entities with int/short PKs); `TestDbContextEnsureCreated` (SQLite + EnsureCreated, users — for `OutboxEvent` which has `long` PK); never mock DbContext.
**frontend auth:** `credentials: 'include'` on all requests; session restored via `GET /auth/session` (returns `{ email, firstName, lastName }`); if session check fails, `AuthContext` explicitly calls `POST /auth/refresh` then retries session check; all HTTP in `features/auth/services/authApi.service.ts`, state in `features/auth/AuthContext.tsx`; no `localStorage`/`sessionStorage` for auth data.
**refresh token:** backend stores opaque tokens in `RTK_RefreshTokens`; `api.service.ts` transparently retries any 401 non-`skipUnauthorized` request via `POST /auth/refresh` (deduplicates concurrent refreshes); `skipUnauthorized: true` disables both the retry and the `unauthorizedHandler`; `silent: true` additionally suppresses the `errorHandler` toast — use for startup probes (`sessionCheck`, `refreshRequest`) that are expected to fail for unauthenticated visitors.
**frontend forms:** all auth forms use `useForm<T>({ resolver: zodResolver(...) })` from React Hook Form; Zod schemas and inferred types live in `src/features/auth/auth.schemas.ts`; per-field errors use `aria-describedby` with field-scoped IDs (e.g. `email-error`); `PasswordInput` uses `forwardRef` so RHF's ref callback reaches the underlying `<input>`.
**frontend CSS:** shared primitives in `@layer components` in `index.css` — extend before adding new classes; brand `brand-600`; cards `bg-white shadow-card border border-slate-200 rounded-2xl`.
**backend validation:** all controller Request DTO validation uses FluentValidation (`AddFluentValidationAutoValidation()` + `AddValidatorsFromAssemblyContaining<Program>()`); `InvalidModelStateResponseFactory` returns `401 UnauthorizedObjectResult(new ErrorResponse { Message = firstError })` to preserve the existing wire format; validators live in `Validators/` and use `CascadeMode.Stop` per-field; `repeatPassword`/`ConfirmPassword` is NOT in backend DTOs — it is a frontend-only UX concern.
**expenses lookup tables:** all enum-like domain values (operation source, family role, rate source, etc.) are DB lookup tables in `Models/Lookups/`, not C# enums; FK columns are `int` (e.g. `CreatedFromId`, `RoleId`); use `ILookupCacheService.GetIdAsync<T>(name)` / `GetNameAsync<T>(id)` to resolve — backed by `IMemoryCache` with `NeverRemove` priority; seed IDs: OperationSource 1=SingleWeb/2=SingleMobile/3=BulkWeb, ModifiedSource 1=Web/2=Mobile, FamilyRole 1=Head/2=Member, RateSource 1=Auto/2=Manual, ConflictStatus 1=Pending/2=Resolved, ConflictResolution 1=AcceptAuto/2=KeepManual/3=Custom, AuditOperation 1=Add/2=Update/3=Delete, SnapshotType 1=Before/2=After.
**expenses controller structure:** mirrors users service — `Controllers/DTO/` for reusable data objects (e.g. `CategoryDto`, `CurrencyDto`), `Controllers/Requests/` for request bodies, `Controllers/Responses/` for envelope responses and `ErrorResponse`; services return DTO types, not model types.
**expenses data provider:** `ExpensesDataProvider` gates all API fetches on `isAuthenticated` from `useAuth()`; fetching without auth causes 401 → transparent refresh → `location.assign('/login')` → infinite reload loop; `isLoading` starts `false` and only becomes `true` during an active fetch.
**frontend tests:** `toHaveClass()` not `toHaveStyle()` — Tailwind doesn't produce inline styles in jsdom.
