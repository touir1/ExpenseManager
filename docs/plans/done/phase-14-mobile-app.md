# Phase 14 — Mobile App (Ionic + Capacitor + React)

## Context

Phase 14 was originally scoped as a PWA. The user wants a **standalone native-quality mobile app** instead — built with Ionic React + Capacitor so it ships as a real iOS/Android APK/IPA while sharing the same backend and as much business logic as possible with the existing web dashboard.

The web dashboard (`frontend/dashboard/`) stays unchanged. The mobile app lives at `frontend/mobile/` as a separate project.

### Push-token backend stub

`POST /notifications/push-token` is added directly to `NotificationController` — no service/repository/migration needed in Phase 14. Phase 15 will add `PushToken` model + FCM/APNs dispatch.

### i18n in Capacitor WebView

`localStorage` works normally inside Capacitor WebView, so `i18next-browser-languagedetector` is kept unchanged. The Settings page writes language preference to both `localStorage` and `@capacitor/preferences` for future native access.

## Goals

- Standalone Ionic + Capacitor + React app at `frontend/mobile/`
- Reuse all API service files, TypeScript types, Zod schemas, and i18n locale JSON from the web dashboard (copy, not symlink — separate build)
- Mobile-first UX: bottom tabs, quick-add modal, day-grouped expense list, swipe-to-delete
- Capacitor plugins for camera (receipt photos), haptics, and push notifications
- Offline add queue via IndexedDB (`idb` library), synced on reconnect
- Same cookie-based auth — Capacitor WebView shares the cookie jar

## Key Mobile Features

### 1. Quick-Add Modal (`QuickAddModal.tsx`)

- `IonModal` with `initialBreakpoint={0.75}` and `breakpoints={[0, 0.75, 1]}` (sheet behaviour)
- Fields: amount (IonInput type=number, autofocus), currency selector, date (IonDatetime), category tiles (IonGrid of IonChip), description (IonTextarea), tags (IonChip input), families (IonSelect)
- Offline: if `useNetworkSync` detects offline, enqueue to IndexedDB instead of calling API
- Haptic feedback (`Haptics.impact`) on submit success
- Same Zod schema + RHF as `ExpenseForm` on web

### 2. Day-Grouped Expense List (`ExpensesListPage.tsx`)

- `IonList` with date-header `IonItemDivider` rows grouped by expense date
- `IonItemSliding` wraps each row → left swipe reveals delete `IonItemOption`
- `IonRefresher` (pull-to-refresh) calls TanStack Query `refetch()`
- `IonInfiniteScroll` for pagination (appends pages as user scrolls)
- Family filter via `IonSegment` at top (matches web `FamilyContext`)

### 3. Mobile Dashboard (`DashboardPage.tsx`)

Simplified vs. web — no recharts needed (use Ionic's built-in or simple SVG):
- `IonCard` with month total + ±% delta
- Category breakdown: `IonProgressBar` per category (top 5)
- Last 5 expenses: `IonList` → taps navigate to edit
- Display currency selector in `IonHeader` toolbar

### 4. Offline Queue (`useOfflineQueue.ts` + `useNetworkSync.ts`)

- `idb` opens store `offline-expense-queue` in IndexedDB
- `useOfflineQueue.enqueue(expenseRequest)` — stores pending add with temp UUID
- `useNetworkSync` listens to `@capacitor/network` `addListener('networkStatusChange')` — on reconnect, drains queue by calling `expensesApi.addExpense()` sequentially; shows toast per success/failure
- Queue item shape: `{ id: string, payload: ExpenseRequest, createdAt: string }`

### 5. Notifications (`NotificationContext.tsx`)

- SignalR connection same as web (cookie auth, `/api/notifications/ws/notifications`)
- Add `@capacitor/push-notifications` registration on app launch — send token to backend (new endpoint needed: `POST /notifications/push-token`)
- On foreground SignalR message: show Ionic toast
- On background: Capacitor push notification (requires backend integration with FCM/APNs — deferred, just register token in Phase 14)

### 6. Receipt Photo (`QuickAddModal.tsx`)

- `IonFab` button inside modal → `Camera.getPhoto({ resultType: CameraResultType.DataUrl })`
- Store base64 string on the expense request body (backend field TBD — not yet in expense schema)
- For Phase 14: store locally, show preview thumbnail; actual backend upload deferred to Phase 15

## API Service Adaptation

`src/services/api.service.ts` needs one change vs. the web copy:

```ts
// Mobile needs an absolute base URL since Capacitor serves from capacitor://localhost
const BASE_URL = import.meta.env.VITE_API_BASE_URL ?? '';
// Then prefix all paths: `${BASE_URL}/api/...`
```

Dev: `VITE_API_BASE_URL=https://localhost` (nginx endpoint, same as web)
Prod: set to actual server URL in `.env.production`

Cookie-based auth works because Capacitor's WebView uses a real browser cookie jar — `credentials: 'include'` behaviour is preserved.
